Glossary

The words this thing uses

Gateway, routing and evidence terms as NautGate means them — not as the industry loosely uses them. Each one maps to something the code actually does.

Attested model

Routing

The model name parsed out of the provider's own response, rather than copied from the request. It is the only field that can be trusted to say what generated the tokens — and it is what makes every other claim on this site checkable, because you can line it up against the provider's billing dashboard token for token.

#evidence#routing

Silent substitution

Routing

When you ask for one model and a different one answers, while the response still claims to be what you asked for. Usually a routing table gap rather than malice — but you cannot detect it by asking the model, because a client's system prompt tells it who to say it is. The flow view highlights the exact hop where the name changed.

#routing#trust

Passthrough lane

Routing

Subscription traffic, recognised by its OAuth token and forwarded to the provider untouched so the plan still covers it. Fully logged, never re-routed. The other lane is metered: an API key that bills per token. Keeping them apart is what lets a Max plan and a routed model coexist without either one quietly paying for the other.

#routing#cost

Model override

Routing

A model pinned to an API key, so every request on that key is served by it regardless of what the client asked for. This is how a client with no model setting of its own gets one — Claude Code has no such switch, but launch it against a pinned key and something else entirely drives the harness.

#routing#clients

DECIDE ladder

Routing

The fixed precedence that picks a model: per-request header, then per-key override, then the caller's stated preference, then the tier score. Every rung is recorded with the decision, so "why did this call go there" is a lookup rather than an argument.

#routing#internals

Tier

Routing

A difficulty band — fast, balanced, deep, expert — scored from the prompt so that model: auto can pick something proportionate instead of always reaching for the biggest thing available. Most requests are not hard; paying frontier prices for all of them is the default waste this is aimed at.

#routing#cost

Decision id

Evidence

The identifier for a single call, joining what was asked, what was decided, what came back and what it cost. The thing to quote in a bug report — it pulls up the whole chain, including the captured prompt and response if policy allowed them to be stored.

#evidence#support

Capture policy

Evidence

The rule deciding whether a prompt or response body is stored at all. Content classified as secret is never written; metadata always is. The classification runs before the write, not after — so the gate is on capture itself rather than on redaction of something already on disk.

#privacy#evidence

Drift

Evidence

A model behaving measurably differently than it did last week — slower, shorter, emptier — detected against its own recorded baseline rather than a vibe. Providers change what sits behind a model name without changing the name, and this is the only way to notice.

#evidence#quality

Head-to-head

Evidence

Real calls grouped by the task they answered, so two models are compared on the same prompt. Averaging a model across all its traffic hides the truth: models get handed different work, and the cheap one often looks better only because it was given easier questions.

#evidence#benchmarks

Shadow trial

Evidence

A cheaper model answering the same prompt in the background, scored blind against the one you actually shipped. Tells you what downgrading would really cost before you commit to it — champion-challenger, run on your own traffic instead of a public benchmark.

#evidence#cost

Subscription saved

Cost

The list price of everything your flat-rate plan covered — what the same work would have cost through the metered API. Notional, not billed: it is the number that tells you whether the plan is earning its keep, and it is usually far larger than people expect.

#cost#reporting

Prefix cache

Cost

Reuse of the work already done on the unchanged front of a prompt — the system prompt and tool schemas that are identical every turn. When it hits, that prefix is near-free; when a prompt changes something early, the whole thing is recomputed. Measured as hit rate, write-to-read ratio, and the dollars saved against the uncached price. Not every local runtime supports it, which is why a model that cannot cache costs far more than its token price suggests.

#cost#cache

Context ratio

Cost

Input tokens burned per output token produced. A high ratio usually means tool schemas rather than thinking — and tool schemas are resent every single turn, so the ratio is where invisible spend shows up first.

#cost#efficiency

Carrying cost

Cost

What your connected tools cost simply by existing — every MCP schema travels in every request whether the model calls it or not. Measured against what those tools save, it is what turns "should I keep this server connected" into an answerable question.

#cost#mcp

Zero egress

Local

No packet leaves the machine — provable by watching its sockets, not by trusting a policy document. The claim a compliance team can actually test: sample the process's connections across a full timer cycle and see nothing but localhost.

#local#compliance

Offline mode

Local

A switch that stands down every timer-driven outbound call — health probes, credit checks, memory sync. Serving traffic is unaffected, so a local model keeps answering while the gateway stops talking to anyone but itself.

#local#airgap

Harness

Local

Everything the client brings — tool implementations, the agent loop, permissions, the system prompt. It belongs to the client, not the model, which is why swapping models underneath a good harness works at all, and why a model's self-report about its own identity is worthless.

#local#agents
